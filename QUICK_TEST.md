# 🧪 木鱼功德机 - 快速测试清单

## ⚡ 立即测试（2分钟）

### 步骤 1：清理和重启（必须）

```
1. 关闭微信开发者工具
2. 等待 5 秒
3. 重新打开开发者工具
4. 导入项目
```

### 步骤 2：编译项目

```
点击 "编译" 按钮
等待编译完成
```

### 步骤 3：检查控制台

打开控制台（Cmd+Shift+C）

**期望看到的输出**：
```
✅ 木鱼功德机已加载
✅ 音频池初始化成功
```

**不应该看到**：
```
❌ Error: timeout
❌ Failed to load
❌ audio load error
```

### 步骤 4：验证显示

**期望**：
```
✅ 大号 🐟 emoji 显示在屏幕中央
✅ "功德累计" 标题显示
✅ "0" 功德值显示
✅ "⚙ 设置" 按钮显示
```

### 步骤 5：点击测试（连续点击 5 次）

**期望每次点击**：
```
✅ 点击立即响应
✅ 木鱼抖动动画
✅ "+1" 浮层向上飘动
✅ 功德值 +1
✅ 音效播放（如果开启）
✅ 震动反馈（如果开启）
```

### 步骤 6：设置测试

点击 "⚙ 设置" 按钮

**测试每一项**：
```
□ 音效开关：关闭 → 开启 → 关闭
□ 震动开关：关闭 → 开启 → 关闭
□ 动画开关：关闭 → 开启 → 关闭
□ 功德增量：+1 → +3 → +9 → +1
```

**期望**：
```
✅ 每个开关正常切换
✅ 设置立即生效
✅ 关闭设置后主界面更新
```

### 步骤 7：数据持久化测试

**操作**：
```
1. 点击木鱼 10 次（确保功德值 > 0）
2. 点击 "⚙ 设置"
3. 点击 "清零功德"
4. 确认清零
```

**期望**：
```
✅ 显示确认对话框
✅ 确认后功德归零
✅ Toast 提示 "已清零"
```

### 步骤 8：重新加载测试

```
1. 点击 "编译" 重新编译
2. 等待重新加载
3. 检查功德值是否保持（如果不清零）
```

## 🐛 问题排查

### 如果测试失败

**问题 1：木鱼不显示**
```
检查：控制台是否有其他错误
解决：清除缓存重新编译
```

**问题 2：点击无响应**
```
检查：控制台错误信息
解决：查看 SOLUTION.md 故障排除
```

**问题 3：音效不播放**
```
检查：audio/knock.wav 是否存在
解决：在设置中关闭音效作为临时方案
```

**问题 4：震动不工作**
```
检查：是否是真机测试
解决：模拟器可能不支持震动，这是正常的
```

**问题 5：数据不保存**
```
检查：控制台是否有 storage 错误
解决：重启开发者工具
```
